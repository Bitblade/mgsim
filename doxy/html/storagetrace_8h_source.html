14 /*
15  A process can write to storages in a cycle.
16  To assist in deadlock analysis, we want to formalize the storages a process
17  can write to. Static code analysis does not give enough information since
18  a process does not have to write to all of its storages in the same cycle.
19  As such, we establish a set of possible storage traces for each process.
20 
21  Since a comprehensive set with all combination of valid traces is tedious
22  to write, we use simple expressions between storages to aid us in creating
23  this set.
24 
25  Possible operations:
26  A ^ B Union. Any from A or B.
27  A * B Sequence. Any from A, then any from B.
28  opt(A) Optional. Any from A or an empty trace.
29 */
